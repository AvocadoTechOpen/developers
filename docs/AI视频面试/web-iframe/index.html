iFrame接入web 端

通过html iframe 接入方便 AI面试 web sdk 包括能力管理,试卷管理,面试报告等页面功能

Examples:​

Examples: https://prod-ai-interview.fbmms.cn/iframe/

Examples为真实线上环境可用的,纯 html+js 实现,代码可以参考.

方案概述​

企业可将自主研发的业务操作页面以 iFrame 的方式嵌入以下功能页面, 适用场景

  1. 解决非 React 技术栈和标准化接入的方式.
  2. 较低的接入成本.免维护,可以使用我们新上线的功能.需同步更新小程序插件.
  3. 适用于不需要定制化的企业.

对接流程​

基础概念​

  1. Access Token: 平台鉴权Token 通过商务提供的 AK SK换取,有一定有效期,所有页面都需要提前设置Access Token 有2个小时的有效期.
  2. 试卷: 由面试题目组成的一个集合.创建成功以后会返回一个 token 作为试卷的唯一标识
  3. 订单: 通过试卷 id 和候选人绑定生成,创建成功以后会返回一个 token 作为订单的唯一标识
  4. 报告id:同订单 id

页面URL​

一共4个 URL

​ 能力模型: https://prod-ai-interview.fbmms.cn/ability

​ - 无参数

​ 创建试卷: https://prod-ai-interview.fbmms.cn/interview

​ - 无参数

​ 编辑试卷: https://prod-ai-interview.fbmms.cn/interview?token={参数}

​ - 创建成功以后的试卷 Token

​ 查看报告: https://prod-ai-interview.fbmms.cn/report?token={参数}

​ - 通过试卷 token和api/standard/b/aiInterview 创建的订单 token

通信方式​

1. url传参: 仅单项传输.对接方传递给我们相关的信息:​

  1. 试卷编辑页面的试卷 token
  2. 获取面试报告需要的 token

2. Postmessage: 双向通信​

  1. 传递给平台的设置 access token
function initAuth() {
    document.getElementById('child')
        .contentWindow
        .postMessage({ type: "setAuth", payload: JSON.parse(localStorage.getItem("auth")) }, "*")
}
  1. 对接方需要获取创建试卷相关信息:

Messages事件:​

取消创建或者编辑试卷:

{
    "type": "onQuestionCancel"
    "playload": {}
}

创建或者试卷成功:

{
    "type": "onQuestionSuccess", 
    "playload": {"token": "k3BGXaSxurS4AXIGHKv3RYV197"}
}
